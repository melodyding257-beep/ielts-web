1.开发环境：
-Claude Code (CC) 安装在 C 盘（全局运行）。
-项目工程、数据（JSON/图片/PDF） 存储在 D 盘。
-Trae 为主编辑器，通过截图或 URL 学习其他网站的 UI 和功能。
2.PPDF 解析逻辑 (0 Token 成本)：
-主体：提取文字版内容，生成“左文右题”的机考布局。
-特殊情况 (扫描版答案/解释)：若检测到答案页是扫描图，只解析基础答案（如 1.A）。
-结果核对：交卷后，如果是扫描版，在页面末尾展示 原 PDF 的截图；如果是文字版答案，则直接显示文字解析，不展示截图。
3.图片解析逻辑：
-支持上传图片，通过 本地 OCR（不花钱）转化为机考练习。
-缺失处理：若图片无答案，提示用户并仅显示文章/题目，不进行判分。
4.用户系统：
-游客模式：可练习，不保存数据。
-登录模式：保存上传材料、做题记录（得分/时间/答案），支持重做。
5.部署与迁移：
-首选 Zeabur 快速上线，后期可随时搬家到 腾讯云/阿里云。
6.技术栈补充：
- 后端：FastAPI(Python)+PyMuPDF(PDF处理)+EasyOCR/PaddleOCR(本地文字图片识别)
- 前端：Node.js + Tailwind CSS(构建雅思机考的高端感)
- 数据库：SQLite
